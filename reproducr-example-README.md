> workflow applied to an analysis of the Palmer Penguins dataset.

---

## What this repo demonstrates

This project shows how `reproducr` integrates into a complete analysis
workflow — from writing a reproducibility-aware script, through risk scoring
and output certification, to automated CI that updates a badge on every push.

| What | How |
|---|---|
| Reproducibility-aware script | `analysis.R` — qualified calls, seeded randomness |
| Risk scoring | `reproducr::audit_script()` + `risk_score()` |
| Output certification | `reproducr::certify()` with committed `.reproducr.rds` |
| Drift detection | `reproducr::check_drift()` on every CI run |
| Auto-updating badge | `reproducr::repro_badge()` in GitHub Actions |
| Audit report | `reproducibility_report.md` — generated by CI |

---

## The analysis

A linear model predicting penguin body mass from bill and flipper measurements,
using the [`palmerpenguins`](https://allisonhorst.github.io/palmerpenguins/)
dataset. The script:

- Uses fully qualified `pkg::fn` calls throughout
- Seeds all stochastic operations with `set.seed(42)`
- Collects 12 key outputs in an `OUTPUTS` list for certification

```r
source("analysis.R")

# Key results
OUTPUTS$n_obs         # 333 complete observations
OUTPUTS$r_squared     # ~0.87
OUTPUTS$cv_rmse_mean  # ~300g cross-validated RMSE
```

## Reproducibility pipeline

### Tier 1 — Scan and score

```r
library(reproducr)

report <- audit_script("analysis.R", verbose = FALSE)
risks  <- risk_score(report)
print(risks)
```

### Tier 2 — Certify and check drift

```r
# On first run — establish the baseline
certify(OUTPUTS, tag = "baseline-v1", script = "analysis.R")

# On subsequent runs — check for drift
check_drift(OUTPUTS, against = "latest")
```

### Tier 3 — Report and badge

```r
repro_report(report, risks, format = "html", style = "pharma",
             output_file = "qc_report.html")

repro_badge(report, risks, output = "README")
```

## CI/CD

The GitHub Actions workflow (`.github/workflows/reproducr-audit.yml`) runs
on every push to `main` and weekly on Monday:

1. Runs `analysis.R` to produce `OUTPUTS`
2. Audits the script with `audit_script()` and `risk_score()`
3. Checks for drift against the last certified run
4. Certifies the current run with today's date as the tag
5. Updates the reproducibility badge in this README
6. Commits `README.md`, `reproducibility_report.md`, and `.reproducr.rds`

The `.reproducr.rds` file is committed to the repository — it is the
auditable history of every certified run.

## Running locally

```r
# Install dependencies
install.packages("palmerpenguins")
pak::pkg_install("reproducr-dev/reproducr")

# Run the analysis
source("analysis.R")

# Audit
library(reproducr)
report <- audit_script("analysis.R", verbose = FALSE)
risks  <- risk_score(report)
print(risks)

# Certify
certify(OUTPUTS, tag = "local-run")

# Report
repro_report(report, risks, format = "html", style = "minimal")
```
